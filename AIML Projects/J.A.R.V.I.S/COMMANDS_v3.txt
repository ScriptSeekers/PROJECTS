HOW TO USE V3
================================================================================

QUICK START (5 STEPS)
────────────────────────────────────────────────────────────────────────────

Step 1: Launch Jarvis
   Command: python v3/jarvis.py
   Wait for: "Jarvis V3 online and ready to assist"

Step 2: Listen for Prompt
   Status: Waiting for voice input
   Audio: "Listening" (spoken by Jarvis)

Step 3: Speak Command
   Format: Natural language
   Speed: Normal conversational pace
   Volume: Clear and distinct
   Example: "What time is it?"

Step 4: Jarvis Processes
   Action: Recognizes command using fuzzy matching
   Speed: ~1-3 seconds processing
   Feedback: Voice response + console output

Step 5: View Results
   Output: Console display
   Response: Voice announcement
   Logging: Automatically saved to database

EXIT PROCEDURE
   Say: "Stop" or "Exit" or "Goodbye"
   Result: 
     • "Thank you for using Jarvis. Goodbye!"
     • Session summary displayed
     • Database saved
     • Program terminates

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

AUDIO INPUT ISSUES
┌─ Problem: Microphone not detected
│  └─ Solution:
│     1. Check microphone connection
│     2. Test in Windows Sound Settings
│     3. Reinstall audio drivers
│     4. Try different USB port
│
├─ Problem: Background noise detected
│  └─ Solution:
│     1. Move to quieter location
│     2. Close nearby applications
│     3. Adjust energy threshold in code
│     4. Use noise-canceling microphone
│
└─ Problem: "Could not request results" (API error)
   └─ Solution:
      1. Check internet connection
      2. Verify Google Speech API is accessible
      3. Check firewall settings
      4. Restart Jarvis

COMMAND RECOGNITION
┌─ Problem: Command not recognized
│  └─ Solution:
│     1. Speak more clearly
│     2. Use command variations (aliases)
│     3. Say "Help" to confirm command exists
│     4. Try different phrasing
│     5. Check COMMANDS_v3.txt for exact syntax
│
├─ Problem: Wrong command executed
│  └─ Solution:
│     1. Speak more distinctly
│     2. Reduce background noise
│     3. Slow down speech rate
│     4. Ensure complete phrase before stopping
│
└─ Problem: Multiple commands in one phrase
   └─ Solution:
      1. Speak one command at a time
      2. Wait for response before next command
      3. Use simple, short phrases

APPLICATION LAUNCH
┌─ Problem: Application won't open
│  └─ Solution:
│     1. Verify application is installed
│     2. Check application is in PATH
│     3. Use alternative application name
│     4. Open manually to verify installation
│     5. Update system PATH variable
│
├─ Problem: Wrong application opens
│  └─ Solution:
│     1. Check application mapping in code
│     2. Use more specific command name
│     3. Modify applications dictionary
│     4. Try alternative command variation
│
└─ Problem: "Application not found"
   └─ Solution:
      1. Install the application
      2. Add to PATH environment variable
      3. Use full executable path in code
      4. Check Windows system PATH

VOICE OUTPUT
┌─ Problem: No audio from Jarvis
│  └─ Solution:
│     1. Check speaker volume
│     2. Unmute system audio
│     3. Reinstall pyttsx3: pip install --upgrade pyttsx3
│     4. Check Windows audio settings
│     5. Restart Jarvis
│
├─ Problem: Voice sounds robotic/broken
│  └─ Solution:
│     1. Adjust speech rate (lower = slower)
│     2. Adjust speech volume
│     3. Install alternative TTS voice
│     4. Update pyttsx3 and dependencies
│
└─ Problem: Speech too fast/slow
   └─ Solution:
      1. Change speech rate in Config class
      2. Range: 50-300 words per minute
      3. Default: 170 (recommended: 150-200)
      4. Lower = slower, Higher = faster

DATABASE ISSUES
┌─ Problem: Database file not created
│  └─ Solution:
│     1. Check write permissions
│     2. Ensure directory exists
│     3. Try different directory
│     4. Reinstall sqlite3
│
├─ Problem: Command history not saved
│  └─ Solution:
│     1. Check database connectivity
│     2. Verify tables are created
│     3. Check for SQL errors in console
│     4. Delete jarvis_data.db and restart
│
└─ Problem: Reminders not triggering
   └─ Solution:
      1. Ensure reminder thread is running
      2. Check reminder time is correct
      3. Check system clock
      4. Monitor background thread status

PERFORMANCE
┌─ Problem: Jarvis is slow/laggy
│  └─ Solution:
│     1. Close unnecessary applications
│     2. Check CPU/memory usage
│     3. Reduce background services
│     4. Check for malware
│     5. Increase system RAM
│
├─ Problem: Commands take long time
│  └─ Solution:
│     1. Check internet connection
│     2. Reduce distance to microphone
│     3. Update all packages: pip install --upgrade
│     4. Check system resources
│
└─ Problem: High memory usage
   └─ Solution:
      1. Restart Jarvis periodically
      2. Clear old database entries
      3. Monitor background threads
      4. Check for memory leaks

BEST PRACTICES
================================================================================

✓ SPEAKING TIPS
  1. Speak clearly and at normal pace
  2. Use complete sentences
  3. Avoid long pauses mid-command
  4. Speak closer to microphone
  5. Reduce background noise
  6. Use command variations if needed

✓ USAGE TIPS
  1. Check command history regularly
  2. Manage reminders to stay organized
  3. Monitor system health periodically
  4. Backup jarvis_data.db regularly
  5. Clear old logs when database grows large
  6. Use "Help" to discover new commands

✓ TECHNICAL TIPS
  1. Update Python packages: pip install --upgrade [package]
  2. Monitor database size: keep under 500MB
  3. Clear old reminders: archive to backup
  4. Test microphone monthly
  5. Restart Jarvis daily for optimal performance
  6. Keep system clean (disk space, RAM)

  1. Backup jarvis_data.db regularly
  2. Keep Python and packages updated
  3. Use strong passwords for API keys

================================================================================
FREQUENTLY ASKED QUESTIONS (FAQ)
================================================================================

Q1: Can Jarvis understand complex sentences?
A: V3 uses fuzzy matching. V4 will have full NLP.

Q2: How accurate is voice recognition?
A: ~95% accuracy in quiet environments. Decreases with noise.

Q3: Can I customize commands?
A: Yes! Modify command handlers in code or add to dictionaries.

Q4: What's the maximum number of reminders?
A: No hard limit. Database grows with each reminder (~1KB per reminder).

Q5: Can Jarvis learn from mistakes?
A: V3 logs all commands. V4 will include learning algorithms.

Q6: Is internet required?
A: Yes, for Google Speech API and weather. Local features work offline.

Q7: Can I run Jarvis on Linux/Mac?
A: Yes! Most features are cross-platform compatible.

Q8: How do I back up my data?
A: Copy jarvis_data.db to safe location regularly.

Q9: Can Jarvis control smart home devices?
A: Not in V3. Planned for V4.

Q10: How do I add new commands?
A: Add to command handler class or create new handler class.
